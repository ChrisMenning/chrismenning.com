Style Guide v1.0

This document defines the complete visual language for chrismenning.com. Every decision is sourced from the actual objects — the gear, the projects, the photographs — not invented from the outside in. Use this as the single reference for all design and development decisions.

Color Usage Rules
→Use #D4A820 TD-3 Gold as the single primary accent across all interactive and highlighted elements.
→Keep LED Green (#50E030) exclusively for status indicator dots. Never use it as a general accent or hover color.
→Maintain the warm-black hierarchy: Void → Panel Black → Panel Dark → Panel Mid for layered depth.
×Never use pure black (#000000) or pure white (#FFFFFF). Every color should carry a warm brown undertone.
×Don't introduce new accent colors. The palette is intentionally closed.
◆Danger Red (#CC2010) appears only in LED indicators and the speed stripe. It is not a general call-to-action or highlight color.

Typography Rules
→Use Oxanium only for the hero name and drop caps. Maximum two Oxanium appearances per page. Its impact depends on rarity.
→All navigation, section numbers, tags, and metadata use Share Tech Mono. This is the "machine voice" — it should feel like a label on a piece of equipment.
→Headings are always uppercase. Body copy is always sentence case. Never mix these within the same level.
×Do not use Oxanium for section headings, card titles, navigation, or any text that appears more than twice on a page.
×Do not use Archivo for labels, tags, or metadata. Mono text and body text must remain visually distinct.
◆Google Fonts load order: Oxanium → Archivo → Share Tech Mono. Archivo is the fallback for Oxanium if the font fails to load — acceptable degradation.

Drop Cap
The ghost outline drop cap is the most distinctive typographic detail on the site. It connects the hero name treatment to the body text — both use Oxanium, both use the outline approach. Use it exactly twice: hero bio and About page intro.

I came to software development in my 30s. The instinct that runs through everything — the home lab, the music, the hardware — is the same: understand how it works, and own what you build.

I work as a software developer and SharePoint admin in Green Bay, writing C# .NET web apps, Salesforce Apex, and untangling permissions issues that nobody else wants to touch. I believe in helping people develop skills — which is how I ended up supporting technical education.

The DIY instinct runs through everything I make. The home lab is built on open source tools I control. The music is made on machines I own outright, with no subscription that can be revoked.

05 // Guidelines
Copy Voice
The design can be undermined by the wrong words. These rules apply to every piece of copy on the site — bio, project descriptions, labels, metadata, everything.
// Do
  • Lead with what the thing is and what it does
  • Name the specific hardware, language, or API
  • Let the breadth speak for itself — don't announce it
  • Describe motivation when it's concrete and specific
  • Write in first person, plainly, sentence case
  • Use dry humor when it appears naturally
  • Keep project descriptions to 2–3 sentences
  • Introduce previous career context only in About
// Don't
  • Lead with KYM or BuzzFeed as a credential anywhere except About
  • Declare values — demonstrate them through the work
  • Use: "passionate", "driven", "innovative", "creative professional"
  • Apologize for or explain the career change in copy
  • Sound bitter about the internet or media industry
  • Over-describe a project — a paragraph, not an essay
  • Write in third person ("Chris Menning is a developer who...")
  • Use exclamation points
Copy Examples
✓ Good — Project description
A repurposed Hanimex Vista Viewer II housing a Raspberry Pi Zero and a small LCD. Queries NASA, iNaturalist, and a carbon countdown API and cycles through them passively. Built because I wanted a screen on my desk that showed something real.
✗ Avoid — Same project, wrong voice
A passion project that showcases my creative approach to hardware hacking and my innovative use of open APIs to create a unique retrofuturistic experience. Demonstrates my ability to blend software and hardware skills.
✓ Good — Bio
I write software, make electronic music on vintage gear, and run a home network that doesn't report back to anyone. I'm a big believer in open source and technology that stays in the hands of the people using it.
✗ Avoid — Bio, wrong framing
After a decade in internet culture and media, I made the difficult decision to retrain as a software developer. Despite a non-traditional background, I am passionate about leveraging technology to create meaningful digital experiences.
06 // Guidelines
Photography
Photography is the most important design element on this site. No stock images, no illustrations, no mockup frames. Only real photographs of real objects. The photos are what make "maker's workshop" feel true rather than claimed.
Surface & Background
The reddish-brown wood desk is the primary backdrop. It appeared in both the Nature Oracle and Tome Reader shots and immediately sets the right warmth and tone. Use it as a consistent surface for product shots. Avoid white, grey, or neutral backgrounds — they strip the warmth that connects the objects to each other.
Lighting
Natural light from the side, not overhead flash. The Nature Oracle photo with the screen lit is the target — the object itself is the light source. Avoid flat, even lighting. Some shadow and gradient gives depth and makes the object feel three-dimensional and present.
Screen State
Always photograph with the screen active and showing real content. The asteroid data on the Nature Oracle is better than a test pattern or blank screen. The manga page in the Tome Reader is better than a startup screen. The content on the screen tells part of the project's story.
Don't Clean Up Too Much
The crime scene tape, masking tape labels, visible USB cables, purple tape on the Tome Reader — these are personality, not flaws. They show the objects are in use. A perfectly clean studio shot would make the work feel finished and distant. Keep the evidence of process.
Shot Types Per Project
Hero shot: straight-on, clean, the object as subject. Context shot: the object in its environment or in use. The cyberdeck on the living room floor with the dog in the background is a perfect context shot — keep that energy. Each project should have both.
Synth Setup
The studio wide shot is good context but needs a clean frame — crop out the floor and focus on the gear surface. Close-up panel shots (the CS01 II, the TD-3 face, the Korg Volca) work especially well as project illustration and visual texture. The colored cable shot from the rack back is site-ready now.

Site Structure
Four pages. Projects is the heart of the site — everything else supports it. The homepage orients, Projects showcases, About explains, Contact connects.
01 // Home chrismenning.com/
// Contains
  • Hero with name (Oxanium display), eyebrow label, bio paragraph with drop cap
  • Status panel — LED indicators, LCD readout, step grid
  • Speed stripe accent
  • Navigation to all sections
// Purpose
  • First impression — polished design sensibility immediately visible
  • Orient the visitor in under 10 seconds
  • Lead with who you are now, not your history
  • Single page — no scrolling content below the fold
02 // Projects chrismenning.com/projects/
// Contains
  • Tag filter bar — All / Software / Music / Hardware / VR / Open Source
  • Asymmetric card grid — featured (7 col) + medium (5 col) + small (4 col)
  • Each card: index number, title, 2–3 sentence description, tags, arrow
  • Individual project pages accessible from cards
// Purpose
  • The heart of the site — everything else supports this page
  • Single pool of work, tagged not siloed — cross-disciplinary projects live naturally here
  • Breadth of skills visible at a glance without requiring a resume
  • Photography carries most of the visual weight
03 // About chrismenning.com/about/
// Contains
  • Longer bio with ghost drop cap on first paragraph
  • Career arc — software work, previous career context, how they connect
  • Home lab section — what runs, why it runs that way
  • Cassette UI — current stack / gear summary
  • Download CV link
// Purpose
  • For visitors who want more after seeing the projects
  • The only place KYM / BuzzFeed context is mentioned
  • Explains the career path without apologizing for it
  • Home lab here as "values with receipts" not a project showcase
04 // Contact chrismenning.com/contact/
// Contains
  • Brief orienting paragraph (1–2 sentences)
  • Email button (primary CTA)
  • GitHub, Bandcamp, LinkedIn, Mastodon links
// Purpose
  • Keep it minimal — don't over-explain here
  • The paragraph sets tone: open to collaborators, employers, music fans
  • No contact form — direct email only
